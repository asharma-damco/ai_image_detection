# Individual Updates — AI Image Detection

One file per person per week (or per day if the project is fast-moving).

## File naming convention
YYYY-MM-DD-[firstname-lastname].md
Example: 2026-05-22-abhishek-sharma.md

## Format per file
# Update — [NAME] | [DATE]
Role: [role on this project]
Period: [date range this update covers]

### What changed
[bullet list of completed work]

### What's in progress
[bullet list of active work]

### Blocked on
[blocker description, or "nothing"]

### Decisions / risks surfaced
[any DEC-XXX or R-XX this update generates]

### Visibility
[internal / stakeholder-visible / codebase-only]

---

## Sources
Updates can come from: people writing them directly, end-of-day pushes via /dev,
meeting notes captured via /capture, or sprint status changes.

Goal: enough signal for the delivery lead to know what moved and where attention
is needed. Not status theatre.
